## 🚀🚀🚀 关于 AELS

**AELS**（Awesome Embedded Learning Studio）是一个围绕真实项目学习嵌入式系统与现代 C++ 的工程笔记工作室，虽然我总是很喜欢把他叫做 AELS 小窝。它由 CharlieChen114514 （也就是笔者）发起，在公开环境下同各位朋友一起持续构建与完善的！我们热烈欢迎任何对系统软件，嵌入式开发，C/C++感兴趣的，在今天浮躁的软件工程实践的今天仍然漫步到这里听我们发发牢骚的朋友们！

您现在在的位置是本组织的中心仓库与官网，也是我们把"动手做东西"过程中产出的工程笔记沉淀下来的小窝的中心处！从这里，向其他任何地方寻找您最喜欢的地方！

## 🧐 虽然带着Awesome，但是这里不是又一个 awesome list

是的，不管您是从哪里来的朋友，相信您看到当今的互联网上，早就出现了不少好的链接合集了！实际上，笔者自己就深受这些Awesome-Lists的影响。直到今天我偶尔都会去翻翻他们，看看他们推荐的一些教学仓库和开源库。

但是，AELS坚决不重复造轮子。我们关注的是那些杰出的合集常常跳过的部分：那些日日夜夜死磕着这些书籍与开源资料中灵光一现**恍然大悟的瞬间**，弯弯扭扭突然昙花一现的**学习路径**，无数次HardFault后找到修复内存践踏后的正确代码，反复评估中最后下定决心的那些**工程取舍**，从左改右改死活无法跑通的设备树到重定向的日志打印出来的Linux 7.1 banner的**喜悦**，这就是我们学习的日常，也是真正的从我们汗水中长出来的教程。知识是死的，他们是活的，知识因为他们而生灵。

笔者必须承认，LLM发展的今天，知识的获取已经极端便宜，甚至可以稍加用心便可以按照自己舒服的方式获取知识。但是，LLM无法代替你学习，特别是嵌入式和系统软件，总是会有一些场景必须您亲自来，这就需要依靠自己从学习生活和工程实践中沉淀的经验处理了。这也是笔者强力回答其他询问我：为什么要做这个的一个重要原因。

这个站范围很大，内容围绕几个方向：嵌入式系统、现代 C++、Linux、操作系统。它是**项目驱动**和主题驱动结合的——大部分写作都从正在做的真实仓库里长出来，并且延伸出来的！

## 📊 截至到落笔的2026年6月24日，这个项目怎么样了？

目前为止，这个组织最成功的项目还是[Tutorial_AwesomeModernCPP](https://github.com/Awesome-Embedded-Learning-Studio/Tutorial_AwesomeModernCPP)，也是反响最好的仓库之一，也是一套正在持续建设的，从 C 和 C++98 到 C++23 的系统化现代 C++ 教程，积累了一些真实关注度。它身边是另一些扎实的工作，比如说以[imx-forge](https://github.com/Awesome-Embedded-Learning-Studio/imx-forge) 为代表的，基于NXP的i.MX6ULL系列的嵌入式Linux BSP和应用层开发教程的仓库，和全新崛起的使用 C++17 开发的Cinux 操作系统系列教程：[Cinux-Book](https://github.com/Awesome-Embedded-Learning-Studio/Cinux-Book)

但是笔者也需要严肃的声明：这些仓库仍然再持续建设和完善。这里不少仓库还在早期、是实验性的，或是正在探索的想法。实际上，从2025年12月底到今天，也才经过半年的时间。大量的内容没有经过时间的考验，仍然需要笔者和各位热心的朋友宝贵的反馈，积极的参与建设。这样，AELS 便能够更好让其他人可以享受到学习，甚至是喜欢上C/C++，嵌入式，系统软件开发这个广泛的领域。

## 🤝 欢迎各位朋友一起建设AELS！

AELS 想做的是一个**C/C++编程语言 + 嵌入式 + 系统软件的开发学习社区**，尽管现在笔者和笔者的另外的朋友们仍然在孤军奋战，我们热烈欢迎您的加入，为其他还在迷茫于陈旧的资料，武断的断言的朋友们，提供一个安心的学习小窝。

无论你是刚入门"Hello, World"的新手，还是有经验的，手撕Linux内核和ARM架构的开发者，我们热烈欢迎您的到来！

您可以随意的在任何仓库上提Issue来报告任何问题：

任何您找到的bug，任何您发现的网站链接死链，任何您觉得这个地方讲的不好，讲的不清楚的地方——甚至不需要反馈怎么不好了，截图，吐槽写的真烂，都是对其他同样正在使用我们项目进行学习的朋友一次巨大的学习体验提升。

任何您发现的错别字，任何您发现的过时的声明，任何您发现跑不起来的地方，任何您发现哪个地方分明需要配一张图才能讲好的，您直接提PR即可！

任何您在相似领域中踩过的坑，做过的玩具，出现过的问题整理成一篇笔记投稿（您不清楚如何做直接开Issue问我们即可，我们会及时反馈！），或者是您有认为合适的项目要推介，我们热烈欢迎！

很多贡献，就是从一个新手提问开始的，从一个文档修正开始的！这也是笔者之前回答过一位朋友如何参与开源项目贡献的回答。

如果感觉不错，欢迎您动动小手点个小⭐！这对我们的工作是一个莫大的认可！感谢您阅读到这里，祝您的学习和开发一路顺利！
